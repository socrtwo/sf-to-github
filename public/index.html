SF2GH Migrator — Help & Instructions

What does this app do?

SF2GH Migrator copies an entire SourceForge project — all branches, all tags, all history — into a new GitHub repository. It works with both Git and SVN projects hosted on SourceForge.

Step 1 — GitHub Token

Paste a GitHub Personal Access Token with the repo scope. This lets the app create repositories on your account and push code to them. Create a token →

Step 2 — Find repos by SourceForge profile NEW

The easiest way to add repos is to look up a SourceForge user:

  1. Paste a SourceForge profile URL into the "Find by SourceForge Profile" field, for example:
    https://sourceforge.net/u/myusername/profile/
    or just type the username: myusername
  2. Click the search icon (or press Enter).
  3. A list of all their projects appears with checkboxes.
  4. Tick the ones you want, then click "Add selected to list".

Step 2 (alternative) — Paste URLs directly

You can also type or paste SourceForge URLs directly into the text area, one per line. Accepted formats:

  • https://sourceforge.net/projects/my-project/
  • https://git.code.sf.net/p/my-project/code — Git only
  • https://svn.code.sf.net/p/my-project/code — SVN only

Step 3 — Options

  • Target Organization — leave blank to migrate to your personal account, or enter an org name (you must be a member with repo-creation rights).
  • Private repositories — tick this to create private repos instead of public ones.

Step 4 — Dry Run vs. Migrate

  • Dry Run (Preview) — shows exactly what the app would do without touching GitHub or SourceForge. Use this first to verify the plan.
  • Migrate to GitHub — creates the GitHub repo(s) and copies all code. This is the real thing.

Modes

  • Server Mode — when running locally with npm start, the app uses your machine's git/svn binaries. Supports both Git and SVN.
  • Browser Mode / Mobile App — when the server is not running (e.g. accessed from a phone or tablet), the app switches automatically to client-side mode using a built-in JavaScript git engine. Git repos work fully; SVN repos require the desktop version.

Status indicator

The dot in the top-right corner shows the current mode:

  • Server v1.x.x — connected to local Node.js server (full Git + SVN support)
  • Browser Mode — client-side mode, server not found (Git only)
  • Mobile App — running as installed iOS/Android app (Git only)
  • Server offline — server not found and browser mode unavailable
